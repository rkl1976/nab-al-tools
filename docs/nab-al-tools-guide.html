01 — Hvad er det?

Et komplet værktøjssæt til AL-udvikling

Når du udvikler apps til Microsoft Dynamics 365 Business Central, skriver du kode i sproget AL. Al brugertekst (knapper, felter, menuer) skal oversættes til flere sprog. Det sker via XLIFF-filer — XML-baserede oversættelsesfiler. NAB AL Tools gør hele denne proces automatisk, fejlfri og AI-assisteret.

✍

XLIFF-oversættelse

Synkroniserer automatisk oversættelsesfiler med din kode. Find uoversatte tekster, match mod Microsofts egne oversættelser, og håndter status for hver enkelt oversættelse.

🤖

AI-integration

Registrerer værktøjer direkte i VS Codes Language Model API og via MCP-serveren. AI-agenter kan oversætte, gennemgå og vedligeholde dine oversættelser autonomt.

📄

Dokumentation

Genererer automatisk markdown-dokumentation for alle AL-objekter med YAML-front matter, indholdsfortegnelser og tooltip-reference.

⚙

CLI & CI/CD

Standalone kommandolinje-værktøjer der kører uden VS Code. Perfekt til build-pipelines, GitHub Actions og automatiserede valideringer.

🔒

Permission Sets

Autogenerer tilladelsessæt for alle AL-objekter og konverter mellem formater. Slut med manuelt vedligehold af tilladelser.

📋

CSV Import/Eksport

Eksporter oversættelser til CSV for ekstern review af oversættere, og importer dem direkte tilbage med korrekt statusmapping.

Baggrund

Hvad er XLIFF, og hvorfor er det vigtigt?

XLIFF (XML Localisation Interchange File Format) er en international standard for udveksling af oversættelsesdata. I Business Central-verdenen fungerer det sådan her:

AL-kode
din kildekode
→
g.xlf
auto-genereret
→
NAB AL Tools
synkroniserer
→
da-DK.xlf / sv-SE.xlf
oversatte filer

Problemet uden værktøjet: Hver gang du ændrer en tekst i din AL-kode, skal alle sprogfiler opdateres manuelt. Nye tekster skal tilføjes, slettede fjernes, og ændrede markeres til genoversættelse. Med 15+ sprog og tusindvis af tekster er det en fejlbombe. NAB AL Tools gør det hele automatisk med én kommando.

// Eksempel: AL-kode med oversættelig tekst
table 50100 "Customer Rating"
{
    fields
    {
        field(1; "Rating"; Integer)
        {
            Caption = 'Customer Rating';
            // ↑ Denne tekst ender i g.xlf og skal oversættes
        }
    }
}

Compileren genererer g.xlf (master-filen) automatisk. NAB AL Tools bruger den som sandhedskilde og synkroniserer alle sprogfiler ud fra den.

02 — Workflow

Fra kode til færdig oversættelse

Hele oversættelsesprocessen følger et klart flow med definerede trin. Her er den typiske arbejdsgang:

1

Skriv AL-kode

Du udvikler tabeller, sider, rapporter og codeunits. Alle brugervendte tekster (Captions, ToolTips, Labels) markeres automatisk som oversættelige af compileren.

.al filer
2

Byg projektet

AL-compileren genererer g.xlf — en masterfil der indeholder alle oversættelige tekster med unikke ID'er og kontekstinformation.

g.xlf Ctrl+Shift+B
3

Synkroniser sprogfiler

NAB: Refresh XLF files from g.xlf — den vigtigste kommando. Sammenligner g.xlf med alle sprogfiler og tilføjer nye tekster, fjerner slettede, og markerer ændrede til genoversættelse.

Refresh XLF da-DK.xlf sv-SE.xlf
4

Oversæt

Nu kan du oversætte på tre måder: Manuelt med "Find next untranslated text" (Ctrl+Alt+U), via AI-agenten der oversætter autonomt, eller via CSV-eksport til eksterne oversættere.

Ctrl+Alt+U AI Agent CSV
5

Match mod base-app

Microsoft har allerede oversat 15+ sprog til Business Centrals standardapp. NAB AL Tools kan automatisk matche dine tekster mod disse officielle oversættelser.

Match Base App Glossar
6

Validér og godkend

Sæt status på oversættelser: translated → signed-off → final. Find inkonsistenser med "Find multiple targets" og sikr kvaliteten.

translated signed-off final
03 — Arkitektur

Hvordan det hele hænger sammen

NAB AL Tools er bygget i lag. Kernelogikken er delt mellem VS Code-udvidelsen, MCP-serveren og CLI-værktøjerne — så den samme gennemtestede kode driver alt.

Indgangspunkter — Tre veje ind
🔌 VS Code Extension

100+ kommandoer, hover-oversættelser, Language Model Tools, webview-editorer. Kræver VS Code.

🤖 MCP Server

Model Context Protocol. Kan bruges af Claude Desktop, GitHub Copilot eller enhver MCP-klient. Kører standalone.

⌨ CLI-værktøjer

RefreshXLF og CreateDocumentation. Til CI/CD-pipelines og automatisering. Ingen UI nødvendig.

deler kode
Delt kerne — Den fælles motor
XliffToolsCore.ts

Fælles implementation for alle AI-værktøjer. Uafhængig af VS Code — kan bruges overalt.

XliffFunctions.ts

1.400+ linjer kernelogik til XLIFF-manipulation: synkronisering, matching, validering.

ALParser.ts

Recursive descent parser for AL-syntaks. Forstår tabeller, sider, codeunits og deres egenskaber.

læser og skriver
Data — Filerne på disken
*.al filer

Din AL-kildekode med Captions, ToolTips og Labels.

g.xlf

Master-oversættelsesfilen. Genereret af AL-compileren. Sandhedskilden.

{sprog}.xlf

Sprogspecifikke oversættelsesfiler: da-DK.xlf, sv-SE.xlf, de-DE.xlf osv.

Glossar (TSV)

Projektspecifikke termer. Sikrer konsistent terminologi på tværs af oversættelser.

Nøgleprincip: CLI-værktøjer og MCP-serveren importerer aldrig VS Codes API. De bruger kun Node.js native API'er. Det verificeres automatisk i CI/CD — så de altid kan køre selvstændigt uden VS Code.

04 — AI-integration

AI der faktisk forstår dine oversættelser

NAB AL Tools eksponerer strukturerede værktøjer som AI-modeller kan bruge. Det betyder at en AI-agent kan læse, oversætte og gemme dine oversættelser — med fuld kontekst om hvert felt.

To veje til AI

🔌 VS Code Language Model API

Værktøjerne registreres direkte i VS Codes chat-interface (Copilot Chat). AI'en kan kalde dem som funktioner under en samtale.

🤖 MCP Server (ekstern)

Samme værktøjer eksponeres via Model Context Protocol. Bruges af Claude Desktop, GitHub Copilot Agent eller andre MCP-klienter — uafhængigt af VS Code.

De 10 AI-værktøjer

getTextsToTranslate
Henter uoversatte tekster med paginering, kildesprog-kontekst, maxlængde og kommentarer. AI'en ved præcis hvad der skal oversættes.
getTranslatedTextsMap
Grupperer oversættelser efter kildetekst. Perfekt til at sikre at "Customer" altid oversættes ens overalt.
saveTranslatedTexts
Gemmer oversættelser og propagerer dem automatisk til alle steder med samme kildetekst. Én oversættelse → alle felter opdateret.
getGlossaryTerms
Returnerer termpar fra BC-glossaret + eventuelt projektglossar. Sikrer at AI'en bruger korrekt terminologi.
getTextsByKeyword
Søg i kilde- og måltekster med regex og paginering. Find alle steder et bestemt ord optræder.
getTranslatedTextsByState
Filtrer oversættelser efter status: needs-review, translated, final. Perfekt til review-workflows.
refreshXlf
Samme synkronisering som kommandoen — men kaldt af AI. Holder filerne opdateret under en AI-session.
buildAlPackage
Bygger AL-projektet og returnerer detaljerede kompileringsfejl med kontekst. AI'en kan debugge build-problemer.
createLanguageXlf
Opretter en ny sprogfil med valgfri matching mod Microsofts base-app oversættelser.
openFile
Åbner filer i VS Code. Etablerer kontekst for de andre værktøjer.

AI-agentens tre tilstande

✍ Oversæt-tilstand
AI'en henter uoversatte tekster batch-vis, oversætter med glossar-kontekst, og gemmer automatisk. Kører autonomt til alt er oversat.
🔎 Review-tilstand
Gennemgår oversættelser der skal godkendes. Markerer som signed-off eller flagger problemer. Batch-baseret review.
📖 Glossar-tilstand
Opretter og vedligeholder projektspecifikke glossarer i TSV-format for konsistent terminologi.
05 — Hvorfor det er smart

De designvalg der gør forskellen

NAB AL Tools er ikke bare "endnu et oversættelsesværktøj". Her er de arkitektoniske beslutninger der gør det til et gennemtænkt og kraftfuldt system:

01

Delt kerne, tre indgange

Den samme gennemtestede kode driver VS Code-udvidelsen, MCP-serveren og CLI-værktøjerne. Fix en bug ét sted — den er fixet overalt. Ingen kode-duplikering.

02

vscode-uafhængighed

CLI og MCP importerer aldrig VS Codes API. Det verificeres automatisk i CI. Resultatet: de kan køre i Docker, i pipelines, på servere — overalt hvor Node.js kører.

03

AI som førsteklasses borger

AI-værktøjerne er ikke et efterfølgende tilføjelse. De er designet med paginering, kontekstrig metadata, og automatisk propagering. AI'en har al den information den behøver for at oversætte korrekt.

04

Base-app matching

Microsoft har oversat tusindvis af termer til 15+ sprog. NAB AL Tools genbruger dem automatisk. Hvorfor oversætte "Customer" til "Kunde" manuelt, når Microsoft allerede har gjort det?

05

Status-workflow

XLIFF-standarden definerer oversættelsesstatus: needs-translation → translated → signed-off → final. NAB AL Tools implementerer hele flowet, så du altid ved præcis hvor langt du er.

06

Propagering

Oversæt "Amount" til "Beløb" én gang — og det opdateres automatisk alle steder i projektet hvor kildeteksten er identisk. Konsistens uden manuelt arbejde.

07

Glossar-system

Kombinerer Microsofts officielle BC-glossar med projektspecifikke termer i TSV-format. AI-agenten bruger glossaret automatisk for korrekt terminologi.

08

CI/CD-klar

RefreshXLF CLI kan validere at alle oversættelsesfiler er synkroniserede som del af din build-pipeline. --fail-changed stopper bygget hvis noget mangler.

09

MCP = universel AI-adgang

MCP (Model Context Protocol) er en åben standard. Det betyder at Claude, Copilot, og fremtidige AI-værktøjer alle kan bruge NAB AL Tools' funktionalitet — uden ændringer i koden.

Opsummering

Et komplet økosystem

NAB AL Tools løser et fundamentalt problem i Business Central-udvikling: håndteringen af flersprogede oversættelser. Det gør det med en arkitektur der er bygget til fremtiden — med AI-integration, CI/CD-support og en delt kerne der sikrer kvalitet og konsistens.
